## 什么是计算机网络

计算机网络是将地理位置不同的、具有独立功能的多台计算机及其外部设备，通过通信线路和通信设备连接起来，在网络协议的管理下实现资源共享和信息传递的系统。

## 网络体系结构

### OSI 七层模型

国际标准化组织（ISO）提出的理论参考模型：

| 层次 | 名称 | 功能 | 协议/设备举例 |
|------|------|------|-------------|
| 7 | 应用层 | 为用户应用提供网络服务 | HTTP、FTP、DNS、SMTP |
| 6 | 表示层 | 数据格式转换、加密解密、压缩 | SSL/TLS、JPEG、ASCII |
| 5 | 会话层 | 建立、管理和终止会话 | RPC、NetBIOS |
| 4 | 传输层 | 端到端的可靠传输 | TCP、UDP |
| 3 | 网络层 | 路由选择与分组转发 | IP、ICMP、ARP、路由器 |
| 2 | 数据链路层 | 相邻节点间的可靠传输 | 以太网、PPP、交换机 |
| 1 | 物理层 | 比特流的透明传输 | 光纤、网线、集线器 |

### TCP/IP 四层模型

实际互联网使用的模型：

| TCP/IP 层次 | 对应 OSI 层次 | 核心协议 |
|-------------|-------------|----------|
| 应用层 | 应用层 + 表示层 + 会话层 | HTTP、DNS、FTP、SMTP、SSH |
| 传输层 | 传输层 | TCP、UDP |
| 网际层 | 网络层 | IP、ICMP、ARP |
| 网络接口层 | 数据链路层 + 物理层 | 以太网、Wi-Fi |

### 五层模型（教学用）

综合 OSI 和 TCP/IP 的优点，常用于教学：

```
应用层 → 传输层 → 网络层 → 数据链路层 → 物理层
```

## 数据封装与解封装

数据在发送端**从上到下**逐层封装，在接收端**从下到上**逐层解封装。

```
应用层：      | 数据 |
传输层：  | TCP头 | 数据 |               → 段（Segment）
网络层：  | IP头 | TCP头 | 数据 |         → 包（Packet）
链路层：  | 帧头 | IP头 | TCP头 | 数据 | 帧尾 |  → 帧（Frame）
物理层：  比特流 010110...
```

**每一层的 PDU（协议数据单元）：**

- 应用层：报文（Message）
- 传输层：段（Segment，TCP）/ 数据报（Datagram，UDP）
- 网络层：分组/包（Packet）
- 数据链路层：帧（Frame）
- 物理层：比特（Bit）

## 各层核心职责

### 物理层

- 定义接口的机械、电气、功能和规程特性
- 解决如何在传输介质上传输比特流
- 关键概念：带宽、波特率、比特率、信道复用（FDM/TDM/WDM/CDM）

### 数据链路层

- 将物理层的比特流组装成帧
- 差错检测（CRC 校验）
- 介质访问控制（MAC）
- 关键概念：MAC 地址、以太网帧、交换机、VLAN

### 网络层

- 负责将数据从源主机送到目的主机（跨网络）
- 路由选择：确定数据包的转发路径
- 分组转发：按路由表转发数据包
- 关键概念：IP 地址、子网掩码、路由协议（OSPF、BGP）

### 传输层

- 提供端到端的通信服务
- TCP：可靠、面向连接、流量控制、拥塞控制
- UDP：不可靠、无连接、高效
- 关键概念：端口号、三次握手、四次挥手、滑动窗口

### 应用层

- 为用户提供各种网络应用服务
- 定义应用进程间通信的规则
- 关键协议：HTTP、HTTPS、DNS、FTP、SMTP、SSH

## 网络设备与工作层次

| 设备 | 工作层次 | 功能 |
|------|---------|------|
| 集线器（Hub） | 物理层 | 信号放大和广播，所有端口共享带宽 |
| 交换机（Switch） | 数据链路层 | 根据 MAC 地址转发帧，各端口独享带宽 |
| 路由器（Router） | 网络层 | 根据 IP 地址转发分组，连接不同网络 |
| 网关（Gateway） | 应用层 | 协议转换，连接不同体系的网络 |

## 常见面试问题

**Q：为什么要分层？**

- 各层独立，降低复杂度
- 灵活性好，每层可独立改进
- 各层可独立实现和测试
- 促进标准化

**Q：OSI 和 TCP/IP 的区别？**

OSI 是理论模型，先有模型后有协议，共 7 层；TCP/IP 是实践模型，先有协议后有模型，共 4 层。实际互联网使用 TCP/IP 模型。

**Q：在浏览器输入 URL 到页面显示，经历了什么？**

1. DNS 解析：域名 → IP 地址
2. TCP 连接：三次握手
3. TLS 握手（HTTPS）
4. 发送 HTTP 请求
5. 服务器处理请求并返回响应
6. 浏览器解析 HTML、加载资源
7. 渲染页面（DOM → CSSOM → 渲染树 → 布局 → 绘制）
8. TCP 断开：四次挥手
