这次实验的题目其实可以拆成一句话：

让不同网段的主机自动拿到 IP 地址，并且在 OSPF 的帮助下实现全网互通。

你给出的要求是：

- 按照拓扑图完成地址规划
- 所有路由器配置 OSPF 路由协议
- 主机采用动态地址分配
- 实验报告要包含拓扑结构图和各路由器的 `running-config`

如果你刚开始学计网，不要一上来就被命令吓住。这个实验的核心不是“背配置”，而是先把下面几件事想清楚：

1. 哪些设备在同一个网段里
2. 每个路由器接口应该配什么 IP
3. 哪些网段需要 DHCP
4. 路由器之间怎样通过 OSPF 知道“去别的网段该往哪走”

## 先把你的拓扑图读明白

从你给的图里，可以拆出 5 个网段：

| 网段 | 用途 |
| --- | --- |
| `36.168.0.0/24` | 中间局域网，连接 `Router1`、`Router2`、`Laptop1`、`Laptop2` |
| `36.168.1.0/24` | `Router0` 和 `Router1` 之间的链路 |
| `36.168.2.0/24` | 左边局域网，连接 `Router0`、`PC0`、`Laptop0` |
| `36.168.3.0/24` | `Router2` 和 `Router3` 之间的链路 |
| `36.168.4.0/24` | 右边局域网，连接 `Router3`、`Laptop3`、`Laptop4` |

注意题目里写的是“`x` 为学号最后两位”。你现在图中写成了 `36.168.x.0/24` 的形式，如果老师要求第一段必须替换成你学号后两位，那你提交前把这里的 `36` 统一换掉即可，方法不变。

## 这些概念分别是什么意思

## IP 地址

IP 地址可以理解成设备在网络里的门牌号。

例如：

- `36.168.2.1`
- `36.168.2.10`

如果两台设备处在同一个网段里，它们通常可以直接通信；如果不在同一个网段，就需要交给路由器帮你转发。

## 子网掩码和 `/24`

你图里所有网段基本都是 `/24`。

`/24` 的意思是：

- 前 24 位表示网络号
- 后 8 位表示主机号

它对应的子网掩码是：

```text
255.255.255.0
```

所以一个 `/24` 网段的规律是：

- 总地址数：`2^8 = 256`
- 可用主机地址数：`256 - 2 = 254`
- 网络地址：最后一段全 0
- 广播地址：最后一段全 255

比如 `36.168.2.0/24`：

- 网络地址：`36.168.2.0`
- 可用范围：`36.168.2.1` 到 `36.168.2.254`
- 广播地址：`36.168.2.255`

## 网段

网段就是“属于同一个网络的一组地址”。

例如：

- `36.168.2.1`
- `36.168.2.20`
- `36.168.2.100`

它们都属于 `36.168.2.0/24` 这个网段。

但 `36.168.3.1` 就属于另一个网段了。

## 默认网关

默认网关就是设备离开本地网段时，第一个要交给它处理的地址，一般就是路由器连接该局域网的接口 IP。

例如在左边局域网 `36.168.2.0/24`：

- 可以把 `Router0` 的 LAN 接口设成 `36.168.2.1`
- 那么 `PC0` 和 `Laptop0` 的默认网关就应该是 `36.168.2.1`

## DHCP

DHCP 的作用是自动给主机分配：

- IP 地址
- 子网掩码
- 默认网关
- DNS 服务器

所以主机不需要手动填地址，只要改成自动获取即可。

## OSPF

OSPF 是一种动态路由协议。

它的作用可以理解成：

- 每台路由器把自己直接连接的网段告诉别的路由器
- 路由器之间自动学习到“去其他网段怎么走”

这样你就不用一条一条写静态路由。

## running-config

`running-config` 就是设备当前正在生效的配置。

在 Cisco 设备里可以用下面的命令查看：

```bash
show running-config
```

实验报告里一般会要求你把每台路由器的 `running-config` 贴出来。

## 这里面的计算怎么做

## 1. 网络地址怎么算

以 `36.168.2.15/24` 为例。

因为 `/24` 表示最后 8 位是主机位，所以把最后一段清零，就是网络地址：

- 主机地址：`36.168.2.15`
- 网络地址：`36.168.2.0`

## 2. 广播地址怎么算

在同一个 `/24` 网段里，把最后一段全变成 `255`：

- 网络地址：`36.168.2.0`
- 广播地址：`36.168.2.255`

## 3. OSPF 通配符掩码怎么算

你在 OSPF 里会看到这种写法：

```bash
network 36.168.2.0 0.0.0.255 area 0
```

这里的 `0.0.0.255` 叫通配符掩码。

它的算法是：

```text
255.255.255.255
-255.255.255.0
=  0.0.0.255
```

也就是：

- 子网掩码：`255.255.255.0`
- 通配符掩码：`0.0.0.255`

因为你的网段都是 `/24`，所以 OSPF 里基本都能统一写成：

```bash
network 36.168.X.0 0.0.0.255 area 0
```

## 这个实验最合理的完成顺序

建议你按这个顺序做，不容易乱：

1. 先给所有路由器接口配置 IP
2. 确保每个接口都 `no shutdown`
3. 先检查直连链路是否正常
4. 再配置 OSPF
5. 再配置 DHCP
6. 最后把主机改成自动获取地址
7. 用 `ping` 和 `show` 命令验收

为什么要这样做？因为如果接口 IP 本身就错了，你后面看到的 OSPF 邻居失败、DHCP 不工作、主机不通，都会混在一起，很难排查。

## Packet Tracer 里应该怎么配

下面给的是一套可以直接照着敲的基础配置。

## Router0

```bash
enable
configure terminal
interface fa0/0
 ip address 36.168.2.1 255.255.255.0
 no shutdown
exit
interface fa0/1
 ip address 36.168.1.1 255.255.255.0
 no shutdown
exit

router ospf 1
 network 36.168.2.0 0.0.0.255 area 0
 network 36.168.1.0 0.0.0.255 area 0
exit

ip dhcp excluded-address 36.168.2.1 36.168.2.20
ip dhcp pool LAN2
 network 36.168.2.0 255.255.255.0
 default-router 36.168.2.1
 dns-server 8.8.8.8
exit
```

## Router1

```bash
enable
configure terminal
interface fa0/0
 ip address 36.168.1.2 255.255.255.0
 no shutdown
exit
interface fa0/1
 ip address 36.168.0.1 255.255.255.0
 no shutdown
exit

router ospf 1
 network 36.168.1.0 0.0.0.255 area 0
 network 36.168.0.0 0.0.0.255 area 0
exit
```

如果你想让中间网段的主机也自动获取地址，可以把 DHCP 放在 `Router1`：

```bash
ip dhcp excluded-address 36.168.0.1 36.168.0.20
ip dhcp pool LAN0
 network 36.168.0.0 255.255.255.0
 default-router 36.168.0.1
 dns-server 8.8.8.8
exit
```

## Router2

```bash
enable
configure terminal
interface fa0/0
 ip address 36.168.0.2 255.255.255.0
 no shutdown
exit
interface fa0/1
 ip address 36.168.3.1 255.255.255.0
 no shutdown
exit

router ospf 1
 network 36.168.0.0 0.0.0.255 area 0
 network 36.168.3.0 0.0.0.255 area 0
exit
```

中间这个 `36.168.0.0/24` 网段同时连接了 `Router1` 和 `Router2`，所以这两台路由器会在这个网段上建立 OSPF 邻居关系。

## Router3

```bash
enable
configure terminal
interface fa0/0
 ip address 36.168.3.2 255.255.255.0
 no shutdown
exit
interface fa0/1
 ip address 36.168.4.1 255.255.255.0
 no shutdown
exit

router ospf 1
 network 36.168.3.0 0.0.0.255 area 0
 network 36.168.4.0 0.0.0.255 area 0
exit

ip dhcp excluded-address 36.168.4.1 36.168.4.20
ip dhcp pool LAN4
 network 36.168.4.0 255.255.255.0
 default-router 36.168.4.1
 dns-server 8.8.8.8
exit
```

## 主机怎么设置

所有 PC 和 Laptop 打开桌面里的 IP 配置页面之后，改成 `DHCP` 或自动获取即可。

对应关系是：

- 左边两台主机从 `36.168.2.0/24` 拿地址
- 中间两台主机从 `36.168.0.0/24` 拿地址
- 右边两台主机从 `36.168.4.0/24` 拿地址

## 为什么不是每台路由器都要配 DHCP

因为 DHCP 是给主机所在的局域网发地址的，不是给路由器之间的点对点链路发地址的。

所以一般只需要在这些主机网段配置 DHCP：

- `36.168.2.0/24`
- `36.168.0.0/24`
- `36.168.4.0/24`

而下面这两条路由器互联链路通常手动写死就行：

- `36.168.1.0/24`
- `36.168.3.0/24`

## 做完之后怎么验收

## 1. 看接口状态

每台路由器都执行：

```bash
show ip interface brief
```

你要看到接口状态和协议状态都正常，常见就是 `up/up`。

## 2. 看 OSPF 邻居

```bash
show ip ospf neighbor
```

正常情况下：

- `Router0` 能看到 `Router1`
- `Router1` 能看到 `Router0` 和 `Router2`
- `Router2` 能看到 `Router1` 和 `Router3`
- `Router3` 能看到 `Router2`

## 3. 看路由表

```bash
show ip route
```

如果配置正确，你会在路由表里看到很多以 `O` 开头的路由，表示这些路由是 OSPF 学来的。

例如在 `Router0` 上，你应该能看到到这些远程网段的路由：

- `36.168.0.0/24`
- `36.168.3.0/24`
- `36.168.4.0/24`

## 4. 看主机是否拿到 DHCP 地址

主机改成自动获取后，要检查它是不是拿到了：

- 正确的 IP 地址
- 正确的子网掩码
- 正确的默认网关

例如左边主机应该拿到 `36.168.2.x`，默认网关应该是 `36.168.2.1`。

## 5. 用 ping 做最终验证

建议你至少测这些：

1. `PC0` ping `Laptop0`
2. `PC0` ping `Laptop1`
3. `PC0` ping `Laptop4`
4. `Laptop3` ping `Laptop1`
5. `Laptop2` ping `36.168.4.1`

如果这些都能通，说明整个实验已经基本完成。

## 最常见的错误

## 忘记 `no shutdown`

这个是最常见的。接口没开，后面所有配置都会表现得像坏了一样。

## 两端 IP 不在同一个网段

例如一边是 `36.168.1.1/24`，另一边却是 `36.168.2.2/24`，那么这两台设备虽然物理连着，也不能正常通信。

## 默认网关写错

主机在 `36.168.2.0/24`，默认网关就应该指向这个网段上的路由器接口，而不是随便填一个别的网段地址。

## OSPF 的 `network` 写错

最容易错的是：

- 网络地址写错
- 通配符掩码写错

你这题网段都是 `/24`，所以统一使用 `0.0.0.255`，已经算比较简单的情况。

## DHCP 地址池包含了网关地址

如果网关已经用了 `36.168.2.1`，就不要再让 DHCP 把 `36.168.2.1` 分给主机，所以要先排除保留地址：

```bash
ip dhcp excluded-address 36.168.2.1 36.168.2.20
```

1. 实验目的

3. IP 地址规划表
4. 配置步骤
5. 各路由器 `running-config`
6. 测试结果
7. 问题与总结

每台路由器都执行：

```bash
show running-config
```

把输出内容整理进实验报告即可。如果老师不要求全部配置，也可以重点放下面三部分：

- 接口配置
- OSPF 配置
- DHCP 配置
