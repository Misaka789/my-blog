拓扑排序是图论里一个很常见、也很实用的主题。

它经常出现在这类问题里：

- 课程安排是否可行
- 任务之间有依赖，如何安排执行顺序
- 编译系统里，哪些模块必须先编译
- 工作流调度中，哪些节点必须先完成

这类题表面看起来不太一样，但核心都一样：**有些事情必须先做，另一些事情才能做**。

只要问题本质是“先后依赖关系”，就很可能可以抽象成拓扑排序。

## 什么是拓扑排序

拓扑排序的作用对象是**有向图**，并且要求这个图是 **DAG**，也就是 **有向无环图**。

如果图中存在一条边 `A -> B`，表示 `A` 必须先于 `B`。

那么拓扑排序要做的事情，就是给图中的所有节点排出一个线性顺序，使得：

- 对于每一条有向边 `A -> B`
- 在最终顺序里，`A` 一定出现在 `B` 前面

比如：

```text
0 -> 1
0 -> 2
1 -> 3
2 -> 3
```

一个合法的拓扑序可以是：

```text
0, 1, 2, 3
```

也可以是：

```text
0, 2, 1, 3
```

因为 `1` 和 `2` 之间没有直接依赖关系，它们的相对顺序不唯一。

## 为什么必须是有向无环图

如果图里有环，就不可能存在合法拓扑序。

例如：

```text
0 -> 1 -> 2 -> 0
```

这意味着：

- `0` 要在 `1` 前面
- `1` 要在 `2` 前面
- `2` 又要在 `0` 前面

这会形成逻辑矛盾，因此不可能排出满足条件的顺序。

所以你可以把拓扑排序理解成一句话：

**拓扑排序本质上是在判断并构造一个符合依赖关系的执行顺序；如果存在环，就说明这种顺序根本不存在。**

## 拓扑排序的两种经典实现

拓扑排序最常见的实现有两种：

1. 基于入度的 BFS，也叫 Kahn 算法
2. 基于 DFS 的后序遍历

面试和刷题里，第一种更常见，也更直观。

## 实现一：基于入度的 BFS

### 核心思想

先找到所有**入度为 0** 的节点。

入度为 0 表示：这个节点没有任何前置依赖，所以它现在就可以被“学习”或“执行”。

然后我们不断重复下面的过程：

1. 从队列里取出一个入度为 0 的节点
2. 把它加入拓扑序
3. 遍历它指向的所有后继节点
4. 将这些后继节点的入度减 1
5. 如果某个后继节点入度变成 0，就把它加入队列

如果最后所有节点都被加入了拓扑序，说明图里没有环；
如果还有节点始终不能变成入度 0，说明图里存在环。

### 为什么这个方法成立

因为一个节点只有在所有前驱节点都已经被处理之后，才会变成入度 0。

也就是说，队列中每次拿出来的节点，一定都是当前“可以合法执行”的节点。

这样逐步推进，得到的顺序自然满足依赖关系。

### 模板代码

下面是一个比较通用的 Java 模板：

```java
import java.util.*;

public class TopologicalSortBFS {
    public List<Integer> topoSort(int n, int[][] edges) {
        List<List<Integer>> graph = new ArrayList<>();
        for (int i = 0; i < n; i++) {
            graph.add(new ArrayList<>());
        }

        int[] indegree = new int[n];
        for (int[] edge : edges) {
            int from = edge[0];
            int to = edge[1];
            graph.get(from).add(to);
            indegree[to]++;
        }

        Queue<Integer> queue = new LinkedList<>();
        for (int i = 0; i < n; i++) {
            if (indegree[i] == 0) {
                queue.offer(i);
            }
        }

        List<Integer> order = new ArrayList<>();
        while (!queue.isEmpty()) {
            int cur = queue.poll();
            order.add(cur);

            for (int next : graph.get(cur)) {
                indegree[next]--;
                if (indegree[next] == 0) {
                    queue.offer(next);
                }
            }
        }

        if (order.size() != n) {
            return new ArrayList<>();
        }
        return order;
    }
}
```

### 时间复杂度

- 建图需要遍历所有边：`O(E)`
- 每个节点最多入队出队一次：`O(V)`
- 每条边最多被访问一次：`O(E)`

总时间复杂度是：

```text
O(V + E)
```

空间复杂度也是：

```text
O(V + E)
```

其中：

- `V` 表示节点数
- `E` 表示边数

## 实现二：基于 DFS

### 核心思想

DFS 的思路不是从“谁现在可以做”出发，而是从“当前节点后面还依赖了谁”出发。

我们对每个节点做深度优先遍历，当它所有后继节点都处理完之后，再把当前节点加入结果。

这相当于一种后序处理。

最后把结果反转，就能得到拓扑序。

### 如何判断有环

DFS 中通常会用一个状态数组：

- `0` 表示未访问
- `1` 表示访问中
- `2` 表示已完成

如果在 DFS 过程中，某个节点访问到了一个“访问中”的节点，说明出现了回边，也就说明图中有环。

### 模板代码

```java
import java.util.*;

public class TopologicalSortDFS {
    private List<List<Integer>> graph;
    private int[] visited;
    private List<Integer> order;
    private boolean hasCycle;

    public List<Integer> topoSort(int n, int[][] edges) {
        graph = new ArrayList<>();
        for (int i = 0; i < n; i++) {
            graph.add(new ArrayList<>());
        }

        for (int[] edge : edges) {
            graph.get(edge[0]).add(edge[1]);
        }

        visited = new int[n];
        order = new ArrayList<>();
        hasCycle = false;

        for (int i = 0; i < n; i++) {
            if (visited[i] == 0) {
                dfs(i);
            }
            if (hasCycle) {
                return new ArrayList<>();
            }
        }

        Collections.reverse(order);
        return order;
    }

    private void dfs(int node) {
        visited[node] = 1;

        for (int next : graph.get(node)) {
            if (visited[next] == 0) {
                dfs(next);
                if (hasCycle) {
                    return;
                }
            } else if (visited[next] == 1) {
                hasCycle = true;
                return;
            }
        }

        visited[node] = 2;
        order.add(node);
    }
}
```

### BFS 和 DFS 怎么选

如果题目要求：

- 判断是否存在合法顺序
- 输出一种合法顺序
- 统计能否完成所有任务

那我一般优先用 **BFS + 入度数组**，因为它更直观，也更贴合“依赖被逐步解除”的过程。

如果题目更强调：

- 图中是否有环
- 深度遍历过程
- 后序构造顺序

那么 DFS 也很适合。

## LeetCode 课程表

题目是 LeetCode 207：**课程表**。

题意可以概括成：

- 一共有 `numCourses` 门课，编号为 `0 ~ numCourses - 1`
- `prerequisites[i] = [a, b]` 表示想学课程 `a`，必须先学课程 `b`
- 问是否能够学完所有课程

### 如何抽象成图

如果 `prerequisites[i] = [a, b]`，含义是：

```text
b -> a
```

也就是：

- `b` 是前置课程
- `a` 依赖 `b`

所以我们建图时应该写成：

- 从 `b` 指向 `a`
- 同时 `a` 的入度加 1

### 这题本质在问什么

这题并不要求你输出完整拓扑序，只要求判断：

**是否存在一种合法的学习顺序，使得所有课程都能学完。**

翻译成图论语言就是：

**这个有向图里是否存在环。**

如果无环，就可以学完；
如果有环，就一定学不完。

### BFS 解法

我们用 Kahn 算法：

1. 建图并统计每个节点的入度
2. 把所有入度为 0 的课程加入队列
3. 不断出队，并把它能解锁的课程入度减 1
4. 统计最后处理了多少门课
5. 如果处理数量等于课程总数，说明可以学完

### Java 代码

```java
import java.util.*;

class Solution {
    public boolean canFinish(int numCourses, int[][] prerequisites) {
        List<List<Integer>> graph = new ArrayList<>();
        for (int i = 0; i < numCourses; i++) {
            graph.add(new ArrayList<>());
        }

        int[] indegree = new int[numCourses];

        for (int[] prerequisite : prerequisites) {
            int course = prerequisite[0];
            int pre = prerequisite[1];
            graph.get(pre).add(course);
            indegree[course]++;
        }

        Queue<Integer> queue = new LinkedList<>();
        for (int i = 0; i < numCourses; i++) {
            if (indegree[i] == 0) {
                queue.offer(i);
            }
        }

        int count = 0;
        while (!queue.isEmpty()) {
            int cur = queue.poll();
            count++;

            for (int next : graph.get(cur)) {
                indegree[next]--;
                if (indegree[next] == 0) {
                    queue.offer(next);
                }
            }
        }

        return count == numCourses;
    }
}
```

## 用一个例子走一遍

例如：

```java
numCourses = 4
prerequisites = [[1,0],[2,0],[3,1],[3,2]]
```

表示：

```text
0 -> 1
0 -> 2
1 -> 3
2 -> 3
```

初始入度：

- `0` 的入度是 0
- `1` 的入度是 1
- `2` 的入度是 1
- `3` 的入度是 2

开始时只有 `0` 能入队。

处理过程如下：

1. 取出 `0`，此时 `1` 和 `2` 的入度都减为 0，入队
2. 取出 `1`，`3` 的入度从 2 变成 1
3. 取出 `2`，`3` 的入度从 1 变成 0，入队
4. 取出 `3`

最后 4 门课程都被处理了，所以答案是 `true`。

如果输入是：

```java
numCourses = 2
prerequisites = [[1,0],[0,1]]
```

图就是：

```text
0 -> 1
1 -> 0
```

这是一个环，没有任何课程能先开始，最终处理数量达不到 `numCourses`，所以答案是 `false`。

## 这道题的易错点

### 1. 边的方向写反

`[a, b]` 表示“学 `a` 之前必须先学 `b`”，所以边是：

```text
b -> a
```

不是 `a -> b`。

这是这道题最常见的错误。

### 2. 以为队列初始只能放一个节点

不是。

所有入度为 0 的节点一开始都应该入队，因为它们都没有前置依赖。

### 3. 只要队列空了就返回 true

这也不对。

队列空了只能说明“当前没有新的可学课程了”，不代表全部课程都处理完了。

正确判断标准是：

```text
处理过的课程数 == numCourses
```

## 一句话总结

拓扑排序处理的是“依赖顺序”问题。

其中最常用的方法是：

- 用邻接表存图
- 用入度数组记录每个节点还剩多少前置条件
- 用队列不断处理入度为 0 的节点

而 LeetCode 课程表，本质上就是在问：

**这个依赖图里有没有环。**

能用拓扑排序处理完所有节点，就说明无环，可以学完；
处理不完，就说明有环，不能学完。
