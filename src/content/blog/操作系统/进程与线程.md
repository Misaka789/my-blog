## 什么是进程

进程是操作系统进行**资源分配和调度的基本单位**。一个运行中的程序就是一个进程。

每个进程拥有独立的：

- **地址空间：** 代码段、数据段、堆、栈
- **文件描述符表**
- **寄存器状态**
- **进程控制块（PCB）：** 操作系统用来管理进程的数据结构

### 进程控制块（PCB）

PCB 是进程存在的唯一标志，包含：

- 进程标识符（PID）
- 进程状态
- 程序计数器（PC）
- CPU 寄存器集合
- CPU 调度信息（优先级、调度队列指针）
- 内存管理信息（页表、段表基址）
- I/O 状态信息
- 记账信息（CPU 时间、实际使用时间）

### 进程状态与转换

进程的五状态模型：

```
新建（New）→ 就绪（Ready）→ 运行（Running）→ 终止（Terminated）
                  ↑               ↓
                  ←—— 阻塞（Blocked/Waiting）
```

- **新建 → 就绪：** 进程创建完成，进入就绪队列
- **就绪 → 运行：** 被调度器选中，分配 CPU
- **运行 → 就绪：** 时间片用完，或被更高优先级进程抢占
- **运行 → 阻塞：** 等待 I/O、等待信号量等
- **阻塞 → 就绪：** 等待的事件完成
- **运行 → 终止：** 进程执行完毕或被终止

## 什么是线程

线程是 CPU 调度的基本单位，是进程中的一个执行流。同一进程的多个线程**共享进程的地址空间和资源**。

每个线程有自己独立的：

- 线程 ID
- 程序计数器（PC）
- 寄存器集合
- 栈空间

共享的资源：

- 代码段
- 数据段
- 堆
- 文件描述符
- 信号处理器

## 进程 vs 线程

| 维度 | 进程 | 线程 |
|------|------|------|
| 定义 | 资源分配的基本单位 | CPU 调度的基本单位 |
| 地址空间 | 独立 | 共享所属进程的地址空间 |
| 创建/销毁开销 | 大 | 小 |
| 切换开销 | 大（需切换页表等） | 小（共享地址空间） |
| 通信方式 | IPC（管道、消息队列等） | 直接读写共享内存 |
| 安全性 | 互不影响 | 一个线程崩溃可能导致整个进程崩溃 |
| 并发性 | 进程间并发 | 进程内并发 |

## 上下文切换

### 进程上下文切换

当 CPU 从一个进程切换到另一个进程时，需要：

1. 保存当前进程的 CPU 上下文（寄存器、程序计数器、栈指针等）到其 PCB
2. 将新进程的上下文从其 PCB 恢复到 CPU
3. **切换页表**（虚拟地址空间切换）
4. 刷新 TLB（Translation Lookaside Buffer）缓存

进程上下文切换的开销较大，主要因为步骤 3 和 4。

### 线程上下文切换

同一进程内的线程切换：

1. 保存当前线程的寄存器、PC、栈指针
2. 恢复目标线程的寄存器、PC、栈指针
3. **不需要切换页表和刷新 TLB**

所以线程切换比进程切换快很多。

## 协程（Coroutine）

协程是一种**用户态的轻量级线程**，由用户程序自己调度，而不是操作系统。

**特点：**

- 在用户态切换，不涉及内核态切换
- 切换开销极小（只需保存/恢复少量寄存器）
- 一个线程上可以运行多个协程
- 协程是协作式调度，需要主动让出 CPU

**与线程的区别：**

| 维度 | 线程 | 协程 |
|------|------|------|
| 调度方 | 操作系统 | 用户程序 |
| 调度方式 | 抢占式 | 协作式 |
| 切换开销 | 中等 | 极小 |
| 并行能力 | 可以多核并行 | 单线程内无法并行 |

## 进程间通信（IPC）

由于进程间地址空间独立，需要特定的 IPC 机制来交换数据。

### 管道（Pipe）

**匿名管道：**

- 半双工通信（数据只能单向流动）
- 只能用于有亲缘关系的进程（父子进程）
- 本质是内核中的一块缓冲区

```c
int fd[2];
pipe(fd);
// fd[0] 读端，fd[1] 写端
```

**命名管道（FIFO）：**

- 以文件形式存在于文件系统中
- 可以在无亲缘关系的进程间通信
- 半双工

```c
mkfifo("/tmp/myfifo", 0666);
```

### 消息队列（Message Queue）

- 消息以链表的形式存储在内核中
- 每条消息有类型（type），可以按类型选择性接收
- 生命周期随内核，不随进程
- 支持随机查询，不一定 FIFO

**优点：** 比管道更灵活，支持消息类型和优先级

**缺点：** 有大小限制，拷贝开销（用户态 ↔ 内核态）

### 共享内存（Shared Memory）

- 多个进程映射同一块物理内存到各自的地址空间
- **速度最快的 IPC 方式**，无需内核拷贝
- 需要配合同步机制（如信号量）使用，否则会有竞态条件

```c
int shmid = shmget(key, size, IPC_CREAT | 0666);
void *ptr = shmat(shmid, NULL, 0);
// 直接读写 ptr 指向的内存
```

### 信号（Signal）

- 用于通知进程某个事件发生
- 是一种**异步通信**方式
- 不能传递大量数据，只能传递信号类型

常见信号：

- `SIGKILL`（9）：强制终止进程
- `SIGTERM`（15）：请求终止进程
- `SIGSTOP`：暂停进程
- `SIGCONT`：继续执行被暂停的进程
- `SIGSEGV`（11）：段错误

### 信号量（Semaphore）

- 本质是一个计数器，用于控制对共享资源的访问
- P 操作（wait）：计数器减 1，若 < 0 则阻塞
- V 操作（signal）：计数器加 1，若 ≤ 0 则唤醒等待进程
- 常与共享内存配合使用

### Socket

- 可以在不同主机之间通信（网络 IPC）
- 也可以在同一主机的不同进程间通信（Unix Domain Socket）
- 全双工
- 支持 TCP/UDP 协议

## IPC 方式对比

| 方式 | 速度 | 数据量 | 方向 | 跨主机 | 适用场景 |
|------|------|--------|------|--------|----------|
| 管道 | 中 | 中 | 半双工 | 否 | 父子进程简单通信 |
| 消息队列 | 中 | 中 | 双向 | 否 | 需要消息类型区分 |
| 共享内存 | 快 | 大 | 双向 | 否 | 大量数据交换 |
| 信号 | 快 | 极小 | 单向 | 否 | 事件通知 |
| 信号量 | - | - | - | 否 | 同步控制 |
| Socket | 慢 | 大 | 全双工 | 是 | 网络通信、跨主机 |

## 线程间通信

同一进程的线程共享地址空间，通信更方便：

1. **共享变量：** 直接读写共享内存，需加锁保护
2. **互斥锁 + 条件变量：** pthread_mutex + pthread_cond
3. **信号量：** sem_wait / sem_post
4. **屏障（Barrier）：** 等待所有线程到达同一点后再一起继续

## 常见面试问题

**Q：为什么线程切换比进程切换快？**

因为同一进程的线程共享地址空间，切换时不需要切换页表和刷新 TLB，只需要保存/恢复寄存器和栈指针。

**Q：多进程和多线程怎么选？**

- 需要高隔离性、安全性 → 多进程（如 Chrome 多进程架构）
- 需要高效通信和共享数据 → 多线程
- 需要极致轻量和高并发 → 协程（如 Go 的 goroutine）

**Q：孤儿进程和僵尸进程？**

- **孤儿进程：** 父进程先退出，子进程被 init（PID=1）进程收养，不会有危害
- **僵尸进程：** 子进程已退出但父进程未调用 `wait()` 回收其 PCB，导致 PCB 残留在系统中。大量僵尸进程会耗尽 PID 资源

**Q：fork() 的返回值？**

- 父进程中返回子进程的 PID
- 子进程中返回 0
- 创建失败返回 -1
